- **Scheduled jobs (`cron_*`)**: Treat **cron_add** as a **binding contract** with the scheduler. **Before** calling it, finish **user clarification** (paths, credentials, recipients, timezone intent, what “success” means). Persist clarified facts through **cron_add**’s structured fields as defined in its tool schema; the job’s **executable body** must run **without a user in the loop** at fire time.
- **Expression**: In the default in-memory scheduler, cron expressions are interpreted in **UTC**; surface that to the user if they assume local time.
